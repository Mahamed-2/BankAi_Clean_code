Core Principle

Clean Architecture

Dependency rule: outer layers depend on inner layers — never the other way around.

Infrastructure
API / Presentation
Application
Domain

Entities · Interfaces · Enums

🧩

Independent of Frameworks

Domain layer has zero references to ASP.NET Core, EF Core, or any library.

🧪

Testable

Every handler is isolated and can be unit tested by mocking IRepository.

🔄

Independent of UI

The Application layer works equally with REST, gRPC, or any delivery mechanism.

🗄️

Independent of Database

Swap SQLite for PostgreSQL/SQL Server by changing one line in Program.cs.

VG Level

Advanced Features

All VG requirements implemented on top of the G-level foundation.

🗺️

AutoMapper + DTOs

Domain entities never exposed directly. MappingProfile maps Product → ProductDto. API contract stays stable even if domain evolves.

Mapping/MappingProfile.cs · DTOs/ProductDto.cs
🔗

MediatR Pipeline Behaviors

ValidationBehavior runs FluentValidation before every handler. LoggingBehavior times execution and warns on slow queries (>500ms).

Behaviors/ValidationBehavior.cs · LoggingBehavior.cs
🔐

JWT Authentication

Register/Login endpoints issue signed JWT tokens with 8-hour expiry. Tokens are validated on every protected request via ASP.NET Core middleware.

Controllers/AuthController.cs · Program.cs (JWT setup)
👑

Role-Based Access Control

Two roles: Admin (full CRUD) and User (read-only). Role is embedded in the JWT claim and enforced via [Authorize(Policy = "AdminOnly")].

Enums/UserRole.cs · Controllers/ (Authorize attributes)
